# imgbase 作業報告書

**日付**: 2026-04-05  
**担当**: masa162  
**目的**: ドメイン移行（img.be2nd.com → img.masa86.com）および定期メンテナンス

## 1. 背景・目的

- be2nd.com の解約方針に伴い、imgbase の配信ドメインを masa86.com に統一する
- 半年ぶりの定期メンテナンス機会として、環境チェック・コードリファクタリングも実施

---

### 判明した課題

1. `imgbase-dev` R2 バケット（preview用）が未作成だった
2. wrangler バージョンが 4.38.0（最新 4.80.0）
3. `SHORT_URL_BASE` が ImageLibrary.tsx にハードコードされていた
4. 日本語ファイル名でアップロードが失敗するバグ（非 ISO-8859-1 文字をヘッダーに直接セット）

---

## 3. 実施作業

### 3-1. コード変更

| ファイル | 変更内容 |
|---------|---------|
| `worker/wrangler.toml` | `PUBLIC_BASE_URL` を `img.masa86.com` に変更、`[[routes]]` に `img.masa86.com/*` を追記 |
| `admin/src/components/ImageLibrary.tsx` | `SHORT_URL_BASE` を `https://img.masa86.com` に変更 |
| `.claude/settings.local.json` | 各種 URL を `img.masa86.com` に更新 |
| `worker/src/index.ts` | ファイル名を `encodeURIComponent` でエンコード（バグ修正） |
| `admin/src/components/ProxyUploadPanel.tsx` | `x-filename` ヘッダーに `encodeURIComponent` を適用（バグ修正） |
| `functions/_middleware.js` | Pages 全体に Basic Auth を適用するミドルウェアを新規作成 |

### 3-2. Cloudflare インフラ操作

| 操作 | 方法 | 結果 |
|------|------|------|
| R2 `imgbase-dev` バケット作成 | `wrangler r2 bucket create imgbase-dev` | ✅ |
| Worker デプロイ（3回） | `wrangler deploy --env=""` | ✅ |
| Worker カスタムドメイン追加 | Cloudflare ダッシュボード手動 | `img.masa86.com` ✅ |
| Worker ルート登録 | wrangler.toml `[[routes]]` 経由 | `img.masa86.com/*` ✅ |
| Pages 環境変数 4件更新 | `wrangler pages secret put` | ✅ |

更新した Pages 環境変数：
- `IMGBASE_UPLOAD_URL` → `https://img.masa86.com/upload/sign`
- `IMGBASE_UPLOAD_COMPLETE_URL` → `https://img.masa86.com/upload/complete`
- `IMGBASE_UPLOAD_PROXY_URL` → `https://img.masa86.com/upload/proxy`
- `IMGBASE_WORKER_URL` → `https://img.masa86.com`
