PM視点でのアドバイス
結論から言うと「小さな摩擦を取り除く」が最も効果的
不満がない = 大きな問題がない = 小さな摩擦にすら気づいていない状態。個人ツールはここが一番伸びしろです。

今すぐやる価値がある（工数小・効果大）
1. クリップボード貼り付けアップロード
スクリーンショット → Ctrl+V → URL コピー の3ステップに。
imageConverter.ts に既に paste イベントの実装があるので、流用するだけ。
これが一番「あ、あってよかった」になる機能です。

2. アップロード時の自動WebP変換
現状: ファイル選択 → コンバーターで変換 → 変換後ファイルを再アップロード（2ステップ）
改善後: ファイル選択するだけでWebPになって保存される（1ステップ）
imageConverter.ts の変換ロジックがそのまま使えます。

3. バックアップ自動化
backup-d1.sh があるのに手動運用。GitHub Actions の weekly cron で自動化するだけ。
「そういえば何ヶ月もバックアップしてない」が永久になくなります。

次々回で検討（工数中・インサイト大）
4. アクセス計測
どの画像が実際にアクセスされているか一切見えていない状態。
D1 に view_count カラムを足して Worker でカウントするだけ。
「誰も見ていない画像」の削除判断ができるようになります。

5. タグ機能（D1スキーマ活用）
実は tags テーブルと image_tags テーブルがD1に既に定義済み。作るだけの状態。
masa86, tokyo86, stk などサイト別に整理できると、ブログ記事メンテ時に「この記事の画像一覧」が出せます。
